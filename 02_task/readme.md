# Порівняння алгоритмів DFS та BFS для знаходження шляхів у графі

У даному завданні продемонстровано роботу двох алгоритмів обходу графів - **DFS** (Depth-First Search / Пошук у глибину) та **BFS** (Breadth-First Search / Пошук у ширину).

Програма знаходить шлях між станціями метро Відня.

## Опис графу 

Граф представляє собою спрощену мережу метро міста Відень (в усіх гілках останні 2-4 станції було відкинуто, так як вони лише зєднані з 2 сусідами). Де вершини - це станції метро, а ребра - переїзди між ними. Граф незважений та неспрямований. 
Візуалізація графа реалізована у пешому завданні.

## Методика тестування
- Проводилось 20 ітерацій порівняння
- У кожній ітерації обирались довільним чином початкова та кінцеві станції
- Обраховувалась кількість ітерацій яка необхідна кожному алгоритму для знаходження маршруту 
- Обраховувалась довжина знайденого маршруту.

## Аналіз результатів

Згідно результатів: алгоритм BFS **завжди** знаходить найкоротший маршрут, проте, для цього, інколт необхідно більше ітерацій алгоритму, а ніж при використанні DFS. 

Шляхи в двох алгоритмів відрізняються через різні стратегії обходу графа:
### BFS (Пошук у ширину)
Досліджує граф покроково (використовується черга FIFO): спочатку всіх безпосередніх сусідів (крок 1), потім сусідів сусідів (крок 2) і т.д. Таким чином BFS гарантовано знаходить **найкоротший** шлях у незважених графах.

### DFS (Пошук у глибину)
Алгоритм обирає одну гілку (використовується Stack) і рухається нею до кінця гілки (або до цілі). У випадку глухого кута алгоритм повертається до останнього розгалужування і продовжує наступною гілкою до кінця гілки (або до цілі). Тому DFS не шукає найкоротший шлях, він шукає **будь-який** шлях. Його мета — відвідати вершини, і якщо перша обрана ним дорога веде до цілі, навіть довгим обхідним шляхом, він поверне саме її.
